# 원격 MCP 서버 연결하기

## 개요

Claude를 원격 MCP 서버에 연결하여 인터넷 호스팅 도구 및 데이터 소스로 기능을 확장하는 방법을 학습합니다.

원격 MCP 서버는 AI 애플리케이션의 기능을 로컬 환경을 넘어 확장하여 인터넷 호스팅 도구, 서비스 및 데이터 소스에 대한 액세스를 제공합니다. 원격 MCP 서버에 연결하면 AI 어시스턴트를 유용한 도구에서 외부 리소스에 대한 실시간 액세스로 복잡한 다단계 프로젝트를 처리할 수 있는 정보가 풍부한 팀원으로 전환할 수 있습니다.

많은 클라이언트가 이제 원격 MCP 서버를 지원하여 광범위한 통합 가능성을 제공합니다. 이 가이드는 [MCP를 지원하는 여러 클라이언트](https://claude.ai/clients) 중 하나인 [Claude](https://claude.ai/)를 예제로 사용하여 원격 MCP 서버에 연결하는 방법을 보여줍니다. Custom Connectors를 통한 Claude의 구현에 초점을 맞추지만, 개념은 다른 MCP 호환 클라이언트에도 광범위하게 적용됩니다.

---

## 원격 MCP 서버 이해하기

### 원격 MCP 서버란?

원격 MCP 서버는 로컬 MCP 서버와 유사하게 작동하지만 로컬 머신이 아닌 인터넷에 호스팅됩니다. Claude가 사용자를 대신하여 작업을 수행하는 데 사용할 수 있는 도구, 프롬프트 및 리소스를 노출합니다.

### 통합 가능한 서비스

- **프로젝트 관리 도구**: Jira, Asana, Trello
- **문서 시스템**: Confluence, Notion, Google Docs
- **코드 저장소**: GitHub, GitLab, Bitbucket
- **API 지원 서비스**: 기타 모든 API 지원 서비스

---

### 원격 MCP 서버의 주요 장점

#### 1. 접근성

- 설치 및 구성이 필요 없음
- 각 장치에서 개별 설정 불필요
- 인터넷 연결이 있는 모든 MCP 클라이언트에서 사용 가능

#### 2. 이상적인 사용 사례

- **웹 기반 AI 애플리케이션**
- **사용 편의성을 강조하는 통합**
- **서버 측 처리 또는 인증이 필요한 서비스**

#### 3. 로컬 서버와의 비교

|특징|로컬 서버|원격 서버|
|---|---|---|
|설치|각 장치마다 필요|불필요|
|접근성|로컬 머신만|인터넷 연결된 모든 곳|
|보안|로컬 제어|서버 측 인증|
|리소스|로컬 파일/데이터|클라우드 서비스/API|
|성능|네트워크 지연 없음|네트워크 지연 있을 수 있음|

## Custom Connectors란?

### 개요

Custom Connectors는 Claude와 원격 MCP 서버 간의 브리지 역할을 합니다. Claude를 워크플로우에 가장 중요한 도구와 데이터 소스에 직접 연결하여 Claude가 즐겨 사용하는 소프트웨어 내에서 작동하고 외부 도구의 완전한 컨텍스트에서 인사이트를 도출할 수 있게 합니다.

### Custom Connectors로 할 수 있는 것

#### 1. 기존 원격 MCP 서버에 연결

- 타사 개발자가 제공하는 원격 MCP 서버에 연결
- [시작 가이드](https://support.anthropic.com/en/articles/11175166-getting-started-with-custom-connectors-using-remote-mcp)

#### 2. 자체 원격 MCP 서버 구축

- 모든 도구와 연결할 수 있는 자체 원격 MCP 서버 구축
- [구축 가이드](https://support.anthropic.com/en/articles/11503834-building-custom-connectors-via-remote-mcp-servers)

## 원격 MCP 서버에 연결하기

원격 MCP 서버에 Claude를 연결하는 프로세스는 [Claude 인터페이스](https://claude.ai/)를 통해 Custom Connector를 추가하는 것입니다. 이는 Claude와 선택한 원격 서버 간에 보안 연결을 설정합니다.

---

### 1단계: Connector 설정으로 이동

**방법:**

1. **브라우저에서 Claude 열기**
2. **설정 페이지로 이동**
    - 프로필 아이콘 클릭
    - 드롭다운 메뉴에서 "Settings" 선택
3. **사이드바에서 "Connectors" 섹션 찾기 및 클릭**

**결과:**

- 현재 구성된 커넥터 표시
- 새 커넥터 추가 옵션 제공

---

### 2단계: Custom Connector 추가

**방법:**

1. **Connectors 섹션에서 아래로 스크롤**
2. **"Add custom connector" 버튼 찾기**
3. **버튼 클릭하여 연결 프로세스 시작**

**대화상자 입력:**

- 원격 MCP 서버 URL 입력 프롬프트 표시
- 서버 개발자 또는 관리자가 제공한 URL 사용
- 완전한 URL 입력 (프로토콜 `https://` 및 필요한 경로 구성 요소 포함)

**예시:**

```
https://example.com/mcp-server
```

**URL 입력 후:**

- "Add" 버튼 클릭하여 연결 진행

---

### 3단계: 인증 완료

대부분의 원격 MCP 서버는 리소스에 대한 안전한 액세스를 보장하기 위해 인증이 필요합니다.

#### 인증 방법

인증 프로세스는 서버 구현에 따라 다르지만 일반적으로 다음을 포함합니다:

- **OAuth**: 타사 인증 제공자를 통한 안전한 권한 부여
- **API 키**: 서버에서 생성된 고유 키
- **사용자 이름/비밀번호**: 전통적인 로그인 자격 증명

#### 인증 프로세스

1. **서버에서 제공하는 인증 프롬프트 따르기**
2. **타사 인증 제공자로 리디렉션될 수 있음**
3. **또는 Claude 내에 양식 표시**
4. **인증 완료 후 Claude가 원격 서버에 보안 연결 설정**

---

### 4단계: 리소스 및 프롬프트 액세스

연결에 성공하면 원격 서버의 리소스와 프롬프트를 Claude 대화에서 사용할 수 있습니다.

#### 액세스 방법

1. **메시지 입력 영역에서 클립 아이콘 클릭**
2. **첨부 메뉴 열기**
3. **연결된 서버의 모든 사용 가능한 리소스 및 프롬프트 표시**

#### 리소스 선택

메뉴에서:

- 대화에 포함할 항목 선택
- 이러한 리소스는 외부 도구의 컨텍스트와 정보를 Claude에 제공
- 여러 리소스 및 프롬프트를 동시에 선택 가능

**사용 가능한 리소스 유형:**

- **문서**: 프로젝트 문서, API 문서
- **데이터**: 데이터베이스 레코드, 분석 데이터
- **코드**: 저장소 파일, 코드 스니펫
- **프롬프트**: 사전 구성된 워크플로우 템플릿

---

### 5단계: 도구 권한 구성

원격 MCP 서버는 종종 다양한 기능을 가진 여러 도구를 노출합니다. Claude가 사용할 수 있는 도구를 제어할 수 있습니다.

#### 권한 구성 방법

1. **Connectors 설정으로 돌아가기**
2. **연결된 서버 클릭**
3. **특정 도구 활성화 또는 비활성화**
4. **사용 제한 설정**
5. **필요에 따라 기타 보안 매개변수 구성**

#### 권한 제어 항목

- **도구 활성화/비활성화**: 특정 도구만 사용하도록 제한
- **사용 제한**: 호출 빈도 또는 데이터 양 제한
- **보안 매개변수**: 민감한 작업에 대한 추가 승인 요구

> **중요**: Claude가 명시적으로 승인한 작업만 수행하도록 보장합니다.

## 원격 MCP 서버 사용 모범 사례

원격 MCP 서버로 작업할 때 안전하고 효율적인 경험을 보장하기 위해 다음 권장사항을 고려하세요:

### 보안 고려사항

#### 1. 서버 인증 확인

- **연결 전 원격 MCP 서버의 진위성 항상 확인**
- 신뢰할 수 있는 소스의 서버에만 연결
- 인증 중 요청된 권한 검토

#### 2. 민감한 데이터 보호

- **민감한 데이터 또는 시스템에 대한 액세스 권한 부여 시 주의**
- 필요한 최소 권한만 부여
- 정기적으로 권한 검토 및 업데이트

#### 3. 보안 체크리스트

- ✅ 서버 제공자의 평판 확인
- ✅ HTTPS 연결 사용 확인
- ✅ 인증 방법 검토
- ✅ 데이터 처리 정책 이해
- ✅ 정기적으로 연결된 서버 감사

---

### 여러 커넥터 관리

동시에 여러 원격 MCP 서버에 연결할 수 있습니다.

#### 조직 전략

**1. 목적별 정리**

```
프로젝트 관리/
├── Jira Connector
├── Asana Connector
└── Trello Connector

개발 도구/
├── GitHub Connector
├── GitLab Connector
└── Docker Connector

문서화/
├── Confluence Connector
├── Notion Connector
└── Google Docs Connector
```

**2. 정기적인 검토**

- 더 이상 사용하지 않는 커넥터 제거
- 작업 공간을 체계적이고 안전하게 유지
- 사용하지 않는 권한 취소

**3. 명명 규칙**

- 명확하고 설명적인 커넥터 이름 사용
- 프로젝트 또는 팀별로 그룹화
- 용도를 쉽게 식별할 수 있도록 라벨 추가

---

### 성능 최적화

#### 1. 리소스 선택 최적화

- 필요한 리소스만 선택
- 큰 데이터 세트는 필터링하여 사용
- 캐싱 활용 (가능한 경우)

#### 2. 네트워크 고려사항

- 안정적인 인터넷 연결 확인
- 대용량 데이터 전송 시 시간 고려
- 오프라인 작업에는 로컬 서버 고려

#### 3. 오류 처리

- 연결 실패 시 재시도 로직 이해
- 타임아웃 설정 확인
- 오류 메시지 로깅 및 검토

## 실전 사용 예제

### 예제 1: 프로젝트 관리 통합

**시나리오:** GitHub와 Jira가 연결된 소프트웨어 개발 팀

**워크플로우:**

1. Claude에게 "이번 스프린트의 진행 상황 요약해줘" 요청
2. Claude가 Jira에서 스프린트 데이터 가져오기
3. GitHub에서 관련 PR 및 커밋 확인
4. 포괄적인 진행 상황 보고서 생성

### 예제 2: 문서 동기화

**시나리오:** Confluence와 Google Docs 연결

**워크플로우:**

1. "API 문서를 Google Docs에서 Confluence로 동기화해줘" 요청
2. Claude가 Google Docs에서 최신 문서 읽기
3. Confluence 형식으로 변환
4. Confluence 페이지 업데이트

### 예제 3: 데이터 분석

**시나리오:** 데이터베이스 서버와 분석 도구 연결

**워크플로우:**

1. "지난 달 판매 데이터 분석해줘" 요청
2. Claude가 데이터베이스에서 데이터 쿼리
3. 통계 분석 수행
4. 시각화 및 인사이트 생성

## 문제 해결

### 일반적인 문제

#### 1. 연결 실패

**증상:**

- 서버에 연결할 수 없음
- 타임아웃 오류

**해결 방법:**

- URL이 올바른지 확인
- 인터넷 연결 확인
- 서버가 온라인 상태인지 확인
- 방화벽 설정 검토

#### 2. 인증 오류

**증상:**

- 로그인 실패
- 권한 거부

**해결 방법:**

- 자격 증명 확인
- API 키 만료 확인
- OAuth 토큰 새로 고침
- 서버 권한 설정 검토

#### 3. 리소스 액세스 실패

**증상:**

- 리소스를 불러올 수 없음
- 빈 결과 반환

**해결 방법:**

- 권한 확인
- 리소스가 존재하는지 확인
- 서버 로그 검토
- 네트워크 상태 확인
